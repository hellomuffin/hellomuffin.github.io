A hands-on guide to Habitat 3.0, Overcooked-AI, and Craftium — the candidate engines for an Interactive Online Policy Evaluation benchmark. Why these three engines

The benchmark needs diverse multi-step interaction tasks where a user physically acts and an assistant could help. The three engines span the fidelity/scale spectrum that question requires — from photorealistic homes to fast 2D coordination to open-ended voxel worlds.

HABITAT 3.0

Photorealistic 3D homes. SMPL-X humanoid + robots, egocentric RGB/depth, real manipulation & articulation (open fridge/drawers). Closest to real wearable-camera deployment.

OVERCOOKED-AI

Real-time 2D coordination. Two cooks, sequential recipe, 50 layouts, bundled human-play data. The cleanest testbed for when to speak & coordination.

CRAFTIUM

Open-world voxel sandbox. Huge compositional action space, first-person RGB, fully Lua-moddable tasks. Best for long-horizon, open-ended assistance.

HABITAT 3.0

Habitat 3.0 — photorealistic embodied homes

▶ Deep dive: the Habitat 3.0 atomic-components reference — scenes (168 houses, 9 examples), objects (16.6k), and an affordance catalog with egocentric videos.

A high-performance 3D simulator (FAIR) with photorealistic scenes, rigid-body physics, articulated objects, and a native SMPL-X human avatar + robots in multi-agent scenes. We drive long-horizon household tasks and render the agent's egocentric camera at every step.

The user body — a walking SMPL-X human

SMPL-X humanoid walking a home ✓

A kinematic human avatar (AMASS walk cycles via HumanoidRearrangeController) tours a furnished apartment. This is the user whose first-person stream the assistant consumes.

Egocentric navigation

First-person RGB through a furnished room — the realistic visual stream the assistant watches.

Social multi-agent & navigation

Social: user + assistant robot ✓

A SMPL-X humanoid (the user) walks the home while a Fetch robot (the assistant) follows and observes — the benchmark's core two-policy setup, in a photorealistic HSSD home.

Goal navigation (ObjectNav-style) ✓

The agent follows the geodesic shortest path to a sequence of goals, emitting the egocentric RGB stream an assistant would watch as the user moves to a destination.

PARTNR — natural-language tasks (100k) ✓

A real PARTNR episode in a photorealistic HSSD home, with the collaborative instruction: “Help me clean up the toys in bedroom. Place all the toy food, pencil case, file sorter, and toy vehicle on the table.” — the language-specified, multi-room household tasks the benchmark ultimately targets.

Multi-step household tasks — continuous egocentric

Each task's grounded PDDL plan (navigate → open cabinet/fridge → pick → place → …) runs with continuous oracle navigation between steps; we capture the robot's head camera every frame — real egocentric video of each distinct task, no keyframe gaps.

OVERCOOKED-AI

Overcooked-AI — real-time 2D coordination

The canonical human-AI coordination testbed (Carroll et al., NeurIPS 2019). Two cooks run a sequential recipe under time pressure: onion → pot ×3 → cook → wait → plate → deliver, looped for reward. We run goal-directed cooks across many layouts — each a different coordination problem.

TaskOrdered soup-delivery loop over the horizon; branching recipes (onion/tomato) & bonus orders.
Actions6/player: N/S/E/W, STAY, context-dependent INTERACT; stepped as a joint action.
AgentsGreedyHumanModel, Random, FixedPlan, trained nets; bundled human-human play datasets.
ObservationsRendered RGB + featurized vector + lossless grid tensor.
Diversity~50 layouts + programmatic generation.
Layouts scoring 0 with two independent greedy cooks (coordination_ring, forced_coordination) isolate exactly the value a proactive assistant adds — resolving coordination the cooks can't alone.
CRAFTIUM

Craftium — open-world voxel sandbox

A Minecraft-like engine (built on Luanti, ICML 2025) with a Gymnasium API, first-person RGB, no Java, and full Lua moddability. Best for a large compositional action space and long-horizon, open-ended tasks. All four below render real first-person video, headless.

TasksBuilt-in: ChopTree, ProcDungeons, Speleo, SpidersAttack, Room, OpenWorld. Custom (authored here as Lua mods): PlantTree, BuildHouse. Plus the open-ended craft/build/survive chain. Lua-extensible; PettingZoo multi-agent.
ActionsRaw: ~21 binary keys (move/jump/sneak/dig/place/drop/inventory/hotbar 1-9) + 2-D continuous mouse look. Per-task Discrete wrappers.
ObservationsFirst-person RGB (configurable res), optional grayscale + 3-D voxel; pos/vel/pitch/yaw in info.
DiversityProcedural worlds & dungeons, day/night, mobs, biomes; the whole Luanti modding ecosystem.
Engineering note. This headless cluster exposes only NVIDIA EGL (no software-GL driver, no X server, no root). Luanti's offscreen GL init is racy here (0–25%/launch, varying with shared-GPU state). We rebuilt the engine from source and added a make_rendering_env retry-on-blank wrapper that recreates the env until a launch renders — which reliably captures every task during good GPU windows.

Run it yourself

Each engine has its own micromamba env under sims/. Representative commands:

# Habitat 3.0
micromamba run -n habitat python sims/habitat/task_hab_oracle.py set_table      # solve a household task
micromamba run -n habitat python sims/habitat/task_humanoid.py                  # SMPL-X user walking
micromamba run -n habitat python sims/habitat/task_wipe_blackboard.py           # custom wipe task

# Overcooked-AI
SDL_VIDEODRIVER=dummy micromamba run -n overcooked python sims/overcooked/task_soup_delivery.py
SDL_VIDEODRIVER=dummy micromamba run -n overcooked python sims/overcooked/showcase.py   # many layouts

# Craftium (retry-on-blank render wrapper)
micromamba run -n craftium-build python sims/minecraft/capture_choptree.py
micromamba run -n craftium-build python sims/minecraft/capture_all.py           # PlantTree / BuildHouse / ProcDungeons

Per-engine research notes & reproduction are in each RESEARCH.md / SETUP.md; a status overview is in sims/TASKS.md.
