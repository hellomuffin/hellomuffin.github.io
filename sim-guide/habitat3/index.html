Atomic components · setup

How we render for the most realistic egocentric view

Habitat ships several 3D scene datasets, and the choice plus the render settings dominate realism. Our findings, with evidence from this repo:

DatasetTypeRealism for egocentric video
HSSD (used here)Artist-authored 3D houses (168 scenes)Best balance — clean watertight geometry, PBR materials, consistent lighting; no reconstruction noise. What Habitat 3.0 itself showcases.
HM3D / MP3DReal-world 3D scansPhoto-real textures but noisy/holey geometry (jagged edges); MP3D is access-gated.

Render settings that matter (measured)

Default settings looked flat and aliased. Enabling these closed most of the gap to the official Habitat 3.0 renders:

  • HSSD scenes instead of test reconstructions — removes jagged walls.
  • 2× supersampling (render at 2× resolution, Lanczos downscale) — anti-aliases edges (Habitat sensors have no MSAA).
  • HBAO ambient occlusion (enable_hbao=True) — adds contact shadows / depth.
  • Head-mounted egocentric camera — the sensor is anchored to the humanoid's actual head link (Habitat's ArticulatedAgentCameraParams.attached_link_id mechanism) so it tracks the head, and it gazes at whatever the hands are doing — a true first-person view.
All scene tours, the object montage, and the manipulation/affordance clips on this page use this HQ recipe (HSSD + HBAO + 2× supersampling).

1.3 Atomic component · most important

Affordances — what the agent can do

Affordances are the actions available on the environment; they define the space of multi-step tasks the benchmark can build. Every clip below is a true first-person (egocentric) view from the humanoid's eyes, pitched down to watch its own SMPL-X hand interact with the object — you can see the whole grasping / opening / closing process, exactly as a wearable head camera would record it. Each loops a single successful execution.

PHYSICALLY-GROUNDED GRASP — Habitat's real constraint API (object obeys gravity)

Pick & drop (real physics)

the object is a DYNAMIC rigid body bound to the hand by create_rigid_constraint (the exact call snap_to_obj makes). It hangs under gravity from the hand, and when the constraint is removed it falls to the floor.

Carry (object swings)

while carried sideways the box swings under its own dynamics on the constraint — not rigidly glued. This is the engine's true grasp behaviour.

These two use Habitat's actual grasp mechanism: a RigidConstraintSettings / RigidConstraintType.Fixed constraint created with sim.create_rigid_constraint(...) between the humanoid's hand link and a dynamic object, stepped with gravity — identical to RearrangeGraspManager.snap_to_obj. The clips below it are the kinematic-pose variants (object follows the hand directly) used for the surface and articulation actions.

Embodiment & camera. A kinematic SMPL-X humanoid (realistic body + hand), driven by HumanoidRearrangeController.calculate_reach_pose, which poses the arm and sets a mocap-derived grasping hand shape as it reaches. The camera is anchored to the humanoid's real head link and gazes at the manipulation point, so it tracks the head like a head-mounted GoPro. 2× supersampled + HBAO. Manipulation runs in HSSD; articulation uses ReplicaCAD's openable furniture (the only Habitat furniture shipping with usable joints).
Honest note on "grasping" in Habitat. Habitat does not simulate fingers physically wrapping an object — by design it uses abstract / "magic" grasp: once the hand is within a threshold, the object is bound to the hand with a rigid physics constraint (RearrangeGraspManager.snap_to_obj → create_hold_constraint), and the SMPL-X hand shows a mocap grip pose. Articulated furniture is opened the same way — a constraint on the handle marker (snap_to_marker) so the joint follows the hand. This is the same grasp abstraction every Habitat task (rearrangement, PARTNR) is built on; there is no dexterous finger-closing in the simulator. The clips above reflect exactly what the engine can express.

5 · the human

Humanoid embodiment & locomotion

7 · dynamics

Physics

Habitat runs the Bullet physics engine: rigid-body dynamics, articulated joints, and constraints. Objects can be STATIC, KINEMATIC, or fully DYNAMIC (gravity, collision, friction).

Rigid-body dynamics

a column of objects falls under gravity, collides, and settles into a pile

Grasp constraint

object held by a rigid constraint hangs under gravity, drops when released

Carry dynamics

the held object swings on the constraint while carried
